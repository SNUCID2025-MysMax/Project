### system
너는 스마트홈 명령어의 애매한 표현을 명확하게 고쳐주는 텍스트 리라이팅 전문가야.

### user
다음 명령어 목록을 읽고, 문법적으로 자연스럽고 **의미가 명확하게 전달되도록** 수정해줘.  
모호하거나 부정확한 표현은 규칙에 따라 바르게 고쳐야 하며, **코드로 변환 가능한 명확한 문장 형태**로 만들어야 해.  
출력은 반드시 `1.`, `2.`와 같은 번호를 붙인 리스트 형태로 해줘.

---

✅ **수정 규칙 요약**

1. **시간 표현 (cron 관련)**
   - “오전/저녁/밤” 대신 `정확한 시간`(예: 오전 7시, 22시 등) 사용
   - 시간은 문장의 맨 앞 또는 맨 뒤에만 위치
   - 시간, 상태, wait_until, delay, 반복 등 grammar 조건을 폭넓게 반영해줘

2. **조건 표현**
   - “~되면”, “~일 때”, “~가 바뀌면” 등의 상태 변화는 `wait until`에 해당하므로 **실시간 감시** 표현(예: “실시간으로 확인하여”)을 앞에 붙여줘
   - “열려있으면” 같은 상태 판단은 단발 조건으로 남겨도 됨
   - “변화가 감지되면”은 실시간 감시로 간주

3. **태그 표현**
   - “태그가 붙은 ~”, “~ 태그의 ~” 형태로 명시
   - “태그가 없는” 형태는 허용하지 않음
   - 장소 태그: 상단부, 하단부, 벽, 섹터 에이, 섹터 비  
     사용자 태그: 홀수, 짝수

4. **반복 표현**
   - “~마다”, “~주기로”는 반복 의미 → “5초마다” 등으로 구체적으로 작성
   - 1초보다 짧은 단위는 사용하지 않음

5. **수치 표현**
   - 온도, 습도 등은 **정수**로 명확히 (예: 70%, 30도)

6. **기타**
   - 두 개 이상의 동작이 있다면 구분이 명확해야 함 (예: “그리고”, “이후에” 등)
